# Definition of Done (DoD)

## 1. Code Review
- Code pushed to `main` must pass all unit tests.
- Code pushed to `main` must merge without conflicts.
- All pull requests must be reviewed by at least one other teammate who did not write the code.
- All review comments must be addressed before merging.

## 2. Manual Testing
- The feature passes all acceptance criteria.
- Black box tests are completed and passed.
- Every acceptance criterion is manually validated during testing.
- All manual test results for the story are documented in the designated project Excel file.
- The team lead confirms that the implementation matches the intent of the story.

## 3. Documentation
- The README and setup instructions are updated if the feature affects deployment or configuration.
- Any assumptions, limitations, or known issues are clearly documented.

## 4. User Experience
- The UI matches approved designs or wireframes (if applicable).
- The feature behaves correctly on all supported browsers/devices.
- Error states and validation messages are implemented and tested.
